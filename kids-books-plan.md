# Digital Kids Books: 1 Book Per Week System

> A systems-first plan using mental models to build a sustainable, SEO-driven
> digital short-book publishing operation for children.

---

## Mental Models Powering This System

| Model | Application |
|-------|-------------|
| **Assembly Line** | Decouple every task (concept → outline → draft → edit → illustrate → publish) so each step runs in parallel or on autopilot |
| **Flywheel Effect** | Book 1 teaches you the template; book 52 writes itself |
| **First Principles** | Strip a kids book to its atoms: 1 concept + 1 character + 1 arc + 5–8 spreads |
| **Minimum Viable Process** | The simplest workflow that ships a quality product weekly |
| **80/20 (Pareto)** | 20% of your niches will drive 80% of traffic — find them in week 1 |
| **Evergreen Stack** | Topics that parents search forever: ABCs, emotions, sleep, friendship, potty |
| **Niche Stacking** | Go narrow first (potty training for toddlers) then wide (toddler life skills series) |
| **Compounding Catalog** | Each published book is a permanent SEO asset that earns while you write the next |

## First Principles: What Is a Digital Kids Book?

Reduced to atoms, every successful kids book contains exactly:

```
1 Core Concept     →  What the child learns or feels
1 Protagonist      →  A child, animal, or object they project onto
1 Emotional Arc    →  Problem → Struggle → Resolution (3 beats)
5–8 Spreads        →  One scene per spread, one sentence per page
1 Takeaway         →  The line a parent reads aloud at bedtime
```

Everything else — art style, rhyme scheme, interactivity — is packaging.
Master the atom first, then dress it up.

## Age-Band Architecture

Three distinct products, each with its own template:

| Band | Ages | Word Count | Spread Count | Reading Level |
|------|------|-----------|--------------|---------------|
| **Board Book** | 0–3 | 50–150 words | 6–8 spreads | 1 noun/page |
| **Picture Book** | 3–6 | 400–800 words | 12–16 spreads | Simple sentences |
| **Early Reader** | 6–8 | 1,500–3,000 words | 20–30 pages | Short chapters |

Start with Picture Books (widest market, easiest to template).
Add Board Books in month 2. Early Readers in month 4.

## SEO Architecture: The Keyword Pyramid

```
        ▲
       / \
      /   \  TIER 1 — 1 per book
     / BIG \  "children's book about anxiety"  (10K–100K/mo)
    /KEYWORD\
   /─────────\
  /           \ TIER 2 — 3 per book
 / LONG-TAIL   \ "picture book to help toddlers with separation anxiety"
/───────────────\  (500–5K/mo — HIGH CONVERSION)
│               │
│  TIER 3       │  TIER 3 — 5 per book
│  MICRO-NICHE  │  "bedtime book for 3 year old with anxiety about daycare"
└───────────────┘  (<500/mo — BUY-INTENT, zero competition)
```

### SEO Title Formula for Every Book

```
[Age] + [Emotion/Topic] + [Format] + [Hook]

Example:  "A Toddler's Book About Worry: Helping Little Hearts Feel Brave"
Example:  "Bedtime for Big Feelings: A Picture Book for Anxious Kids Ages 3-6"
```

### Metadata Stack Per Book

Every book ships with:
- **Title**: SEO formula above
- **Subtitle**: secondary keyword phrase
- **Description**: 250-word keyword-rich blurb (parent-facing problem → solution)
- **7 Keywords** (Amazon/platform): exact match search terms
- **BISAC Codes**: Juvenile / Social Situations + Emotions & Feelings
- **Back Matter**: author page + 3 related book recommendations (internal links)

## The Weekly Production System

### Time Budget: 8–10 hours/week

```
Monday      2h   CONCEPT + OUTLINE
Tuesday     2h   FIRST DRAFT (full text)
Wednesday   1h   EDIT + READ ALOUD TEST
Thursday    2h   ILLUSTRATION PROMPTS + AI IMAGE GENERATION
Friday      2h   LAYOUT + PUBLISH + SEO METADATA
Saturday    30m  SOCIAL ASSET CREATION (1 reel, 3 posts)
```

### Monday: Concept + Outline (2h)

**Step 1 — Keyword research (30 min)**
Pull 3 candidate topics from your Pillar rotation.
Check search volume on: Amazon search bar, Google autocomplete,
Pinterest search, Teachers Pay Teachers trending.

**Step 2 — Concept validation (15 min)**
Ask: Does this topic make a parent stop scrolling?
Does it solve a pain they feel today?
Is there a clear child protagonist moment?

**Step 3 — Fill the Story Blueprint (45 min)**

```
STORY BLUEPRINT TEMPLATE
─────────────────────────
Title (working):
Age Band:
Core Concept (1 sentence):
Protagonist (name + 1 trait):
Problem (opening spread):
3 Attempts to solve (spreads 2–4):
Turning point (spread 5):
Resolution (spread 6):
Takeaway line (final spread):
Parent note (back matter, 2 sentences):
```

**Step 4 — Spread-by-spread outline (30 min)**
Map each spread: Scene | Character action | Text (rough)

---

### Tuesday: First Draft (2h)

Use your **Voice Guide** (built once in week 1, reused forever):

```
VOICE GUIDE
───────────
Sentence length:   Max 12 words per sentence
Vocabulary:        Substitute any word a 4-year-old wouldn't know
Rhythm:            Read aloud every 3 sentences. Does it breathe?
Emotion words:     Name the feeling explicitly at least once
Repetition device: Use a refrain (e.g., "And that was okay.")
Ending tone:       Warm + concrete + hopeful
```

Draft rule: **Write ugly first.** No editing during draft. Speed matters.
Target: complete text in 90 minutes, 30 minutes for tightening.

---

### Wednesday: Edit + Read Aloud Test (1h)

**The 4-Pass Edit System:**

| Pass | Focus | Tool |
|------|-------|------|
| 1 — Structural | Does the arc land? | Outline vs. draft |
| 2 — Sentence | Every sentence earns its place | Read aloud |
| 3 — Voice | Consistent tone, age-appropriate words | Voice guide checklist |
| 4 — Parent Layer | Is there a second-level meaning adults value? | Adult read-through |

**Read Aloud Test:** Record yourself reading it to a timer.
Picture book should land in 3–5 minutes at a natural parent pace.
If you stumble, the sentence is broken.

---

### Thursday: Illustration + Layout Prep (2h)

**Illustration Prompt Template (per spread):**

```
Spread [N] Illustration Prompt:
Style: [watercolor / flat vector / soft pencil — pick ONE for entire book]
Scene: [what's happening, described in 1 sentence]
Character: [protagonist + emotion + body posture]
Background: [setting, time of day, 2–3 key elements]
Color palette: [warm/cool, 3 hex codes or descriptors]
Mood: [the feeling this spread must convey]
Text space: [top / bottom / left / right — leave clear for text overlay]
```

**AI Image Workflow:**
- Generate 3 variants per spread prompt
- Select best, note the seed for consistency
- Keep a **Character Bible** (seed + prompt settings) for series books

**Character Bible Template:**

```
CHARACTER: [Name]
Base prompt: [exact prompt string that generates them consistently]
Seed/settings: [model, seed number, cfg, steps]
Colors: [hair, skin, clothes hex codes]
Age visual: [appears ~X years old]
Signature item: [the one prop they always carry]
```

---

### Friday: Layout + Publish + SEO (2h)

**Layout Tools (pick one, stick with it):**
- Canva Pro (fastest, best templates, export to PDF/ePub)
- Adobe Express (more control)
- Book Bolt (Amazon KDP optimized)

**Layout Checklist:**
- [ ] Trim size set: 8.5 × 8.5 (square = most versatile)
- [ ] Bleed set: 0.125" all sides
- [ ] Font: Max 2 fonts (display + body). Min 24pt body for ages 3–6
- [ ] Text contrast passes WCAG AA on all backgrounds
- [ ] Page numbers hidden on spreads, visible on back matter
- [ ] ISBN block placed (back cover, bottom right)
- [ ] Copyright page complete
- [ ] Back matter: Author bio + 3 book recommendations

**Publish Checklist:**
- [ ] Amazon KDP (primary — largest reach)
- [ ] Payhip or Gumroad (direct sales, 0% fee, instant delivery)
- [ ] Google Play Books
- [ ] Draft2Digital → aggregates to 40+ retailers automatically

---

### Saturday: Social Content (30 min)

**The 1-Reel / 3-Posts Formula:**

From each book, create once, distribute everywhere:

```
Reel (60s):  "Here's why I wrote a book about [topic]" + 3-page flip
Post 1:      The takeaway line as a typographic quote card
Post 2:      Behind-the-scenes: spread layout or character sketch
Post 3:      Parent tip related to the book's topic (education content)
```

**Caption Formula:**
```
Hook line (problem the parent recognizes)
↓
The book solves it because [specific reason]
↓
"Link in bio to grab your copy — also free on KU"
↓
3 hashtags: #[topic]booksfortoddlers #[ageBand]reads #[emotion]books
```

## The Book Backlog System

Never run out of ideas. Build a 52-book backlog before week 1.

**The Backlog Generator:**

```
For each of 4 Pillars × 3 Age Bands = 12 Slots
Fill each slot with 4–5 specific story concepts
= 48–60 titles in the pipeline
```

**Backlog Entry Format:**

```
ID:         P1-PB-003
Pillar:     Emotional Intelligence
Age Band:   Picture Book (3–6)
Topic:      Worry about the first day of school
Title idea: "What If My Tummy Butterflies Never Stop?"
Hook:       Every child + every parent knows this moment
Season:     August (back-to-school — publish July)
Status:     [ ] Queued  [ ] In Progress  [ ] Published
```

**Seasonal Calendar Layer:**

| Month Published | Topic Cluster |
|-----------------|---------------|
| July–Aug | Back to school anxiety, new teacher, new friends |
| Oct–Nov | Feelings about holidays, gratitude, sharing |
| Dec–Jan | Family changes, new year emotions |

## Revenue Architecture

### Three Revenue Streams, One Catalog

```
STREAM 1 — Royalties (passive, compounding)
  Amazon KDP: 70% royalty on $2.99–$9.99 ebooks
  KDP Unlimited: per-page read rate (~$0.004/page)
  D2D stores: 60–65% net

STREAM 2 — Direct Sales (highest margin)
  Payhip: sell PDF/ePub at full price, keep 97%+
  Bundle 3–5 related books: $9.99–$14.99 bundle
  Series bundles compound: book 1 at $0.99 → upsell to series

STREAM 3 — Licensing + Derivatives
  Teachers Pay Teachers: activity packs per book (+$4.99)
  Classroom licensing: school district bulk deals
  Print-on-demand: same files → KDP paperback, $0 incremental work
```

| Format | Price | Platform |
|--------|-------|----------|

| Series bundle (3 books) | $9.99 | Payhip |

```
Write Book 1
    ↓
Learn what sells (keyword, pillar, age-band)
    ↓
Build template from what worked
    ↓
Write Book 2 faster, better targeted
    ↓
Catalog grows → internal links multiply → SEO compounds
    ↓
Series audiences form → email list grows
    ↓
Email list pre-launches new books → launch day spike → Amazon rank boost
    ↓
Rank boost → organic discovery → new readers
    ↓  ↑

```

**The flywheel accelerates when:**
- You publish series (same character, 4+ books) — readers binge
- You capture emails at book 1 → auto-promote books 2–N
- Back matter links every book to 3 others in catalog

## Quality Control System

One checklist before every publish:

```
PUBLISH GATE CHECKLIST
──────────────────────
[ ] Read aloud time: 3–5 minutes (picture book)
[ ] Child test: Read to 1 actual child in target age band
[ ] Parent test: Does a parent want to read this twice?
[ ] Arc check: Problem → struggle → resolution all present
[ ] Takeaway: Last line is memorable and warm
[ ] SEO title: Contains primary keyword naturally
[ ] Description: 250+ words, solves a named parent problem
[ ] Images: Consistent character, consistent style, no text errors in art
[ ] Layout: All text readable, bleeds correct, PDF exports cleanly
[ ] Back matter: Author bio + 3 book links present
[ ] Copyright page: Year, author name, ISBN
```

## Week 1 Action Plan

Before writing a single word, build the infrastructure:

1. **Day 1** — Create your Story Blueprint template, Voice Guide, Character Bible template in Notion/Obsidian
2. **Day 2** — Do keyword research for all 4 pillars. Fill 52-slot backlog
3. **Day 3** — Set up KDP account, Payhip store, Draft2Digital account
4. **Day 4** — Set up Canva Pro with one 8.5×8.5 master template
5. **Day 5** — Set up Midjourney. Run 10 test prompts. Choose art style. Lock character bible
6. **Day 6** — Write and publish Book 1 using the full system (this IS your workflow test)
7. **Day 7** — Retrospective: What took longest? Optimize that step first

**The only goal of Week 1 is to publish one imperfect book and learn the system.**
The system will compound. The first book is the template.
